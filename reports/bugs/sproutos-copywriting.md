<br>

🔍 **Issue**
Eleven UI feedback strings are written as plain JavaScript string literals. They are never passed through WordPress's `__()` translation function, so they cannot be translated. Any site running a non-English locale will always display raw English text in these states.

<br>

📋 **Steps to Reproduce**
1. Open `sprout-core/admin-pages.php`
2. Search for each string listed in the table below
3. Change the WordPress site language to any non-English locale (e.g., French or German)
4. Trigger the corresponding UI state — copy action, network error, log purge, etc.
5. Observe the string still appears in English regardless of locale

<br>

❌ **Current Output**

| Line | Hardcoded String |
|:---|:---|
| 1027 | `'Error'` |
| 1029 | `'Network error. Please try again.'` |
| 2038 | `'Copied!'` |
| 2042 | `'Error copying'` |
| 3210 | `'Loading...'` |
| 4297 | `'Failed to load details.'` |
| 4305 | `'Failed to purge logs.'` |
| 4306 | `'Logs purged.'` |
| 4787 | `'Loading...'` |
| 4819 | `'Failed to load details.'` |
| 5097 | `'Error'` |

<br>

✅ **Expected Output**
Each string is localised and automatically respects the active site language.

<br>

🛠️ **Solution**
- Declare each string as a PHP-localised variable above the inline script block
- Use `wp_json_encode( __( 'string', 'sproutos' ) )` pattern for every string
- Replace the raw JS string with the PHP variable in all `flash()`, `innerHTML`, and similar calls
- Apply this pattern as the standard for every string listed above:

```php
var errorMsg   = <?php echo wp_json_encode( __( 'Error', 'sproutos' ) ); ?>;
var networkErr = <?php echo wp_json_encode( __( 'Network error. Please try again.', 'sproutos' ) ); ?>;
var copiedMsg  = <?php echo wp_json_encode( __( 'Copied!', 'sproutos' ) ); ?>;
```
